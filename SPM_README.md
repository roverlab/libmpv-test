# Swift Package Manager Configuration

This project has been updated to use Swift Package Manager (SPM) for better dependency management and to resolve Xcode project file corruption issues.

## Project Structure

```
.
├── Package.swift              # SPM configuration
├── Sources/
│   └── Libmpv/
│       ├── include/
│       │   ├── Libmpv.h       # C header exports
│       │   └── module.modulemap # Module map for C libraries
│       ├── Libmpv.m           # C wrapper implementation
│       └── Libmpv.swift       # Swift interface
├── Tests/
│   ├── MPVScreenshotTests.swift # Test cases
│   └── Info.plist            # Test configuration
└── lib/                      # Pre-built static libraries (generated by build.sh)
```

## Building with SPM

### Prerequisites

1. First, build the native libraries using the existing script:
   ```bash
   ./download.sh
   ./build.sh -e development
   ```

2. This will create static libraries in the `lib/` directory.

### Using SPM

#### Option 1: As a Local Package

```bash
# In your project directory
git clone https://github.com/your-org/libmpv-test.git
cd your-project
swift package init
# Add dependency to Package.swift
```

#### Option 2: Direct Integration

Create a `Package.swift` in your project:

```swift
// swift-tools-version: 5.9
import PackageDescription

let package = Package(
    name: "YourApp",
    platforms: [.iOS(.v13)],
    dependencies: [
        .package(path: "./libmpv-test") // Local path
    ],
    targets: [
        .target(
            name: "YourApp",
            dependencies: ["Libmpv"]
        )
    ]
)
```

### Building with Xcode

```bash
# Generate Xcode project
swift package generate-xcodeproj

# Or open directly in Xcode 15+
xed Package.swift
```

## Running Tests

### Local Testing
```bash
# Run tests with SPM
swift test

# Run specific test
swift test --filter MPVScreenshotTests
```

### CI/CD Integration

The main CI workflow (`.github/workflows/build-ios.yml`) now uses SPM testing directly, replacing the old xcodebuild approach.

Tests run automatically on:
- Push to main/master/develop branches
- Pull requests to main/master  
- Manual workflow dispatch

No separate SPM workflow is needed - everything is integrated into the main build process.

## Troubleshooting

### Library Not Found Errors

If you get "library not found" errors:

1. Ensure you've run `./build.sh -e development`
2. Check that static libraries exist in `lib/` directory:
   - `libmpv.a`
   - `libavformat.a`
   - `libavcodec.a`
   - `libavutil.a`
   - etc.

### Header Search Path Issues

The package assumes C library headers are available. If you built with `build.sh`, they should be in the SDK path.

### Architecture Mismatches

For iOS Simulator (x86_64) vs Device (arm64):

```bash
# Development (includes both architectures)
./build.sh -e development

# Distribution (device only, optimized)
./build.sh -e distribution
```

## Migration from Old Xcode Project

1. **Backup your existing project**
2. **Delete the corrupted `.xcodeproj`**
   ```bash
   rm -rf Tests/MPVTestApp.xcodeproj
   ```
3. **Use SPM generated project**
   ```bash
   swift package generate-xcodeproj
   ```
4. **Or use Xcode 15+ native SPM support**
   - File → Open → Select `Package.swift`
